# Copilot Quality Checklist

This checklist should be reviewed before completing any code changes.

## CSS/Styling Checklist
When adding new UI elements or modals:

- [ ] **Dark theme styles added** - All new elements have dark theme CSS
- [ ] **Light theme styles added** - All new elements have `body[data-theme="light"]` CSS
- [ ] **All input types covered** - Include text, password, number, select, textarea, etc.
- [ ] **Focus states defined** - All interactive elements have :focus styles
- [ ] **Hover states defined** - All clickable elements have :hover styles
- [ ] **Responsive design** - Mobile breakpoints considered (@media queries)
- [ ] **Color consistency** - Colors match existing theme palette

## Form Elements Checklist
When creating forms:

- [ ] **All input types styled** - text, number, select, checkbox, radio, etc.
- [ ] **Labels properly associated** - for/id attributes match
- [ ] **Placeholder text added** - Where appropriate
- [ ] **Required fields marked** - Required attribute set
- [ ] **Form validation** - Client-side validation logic added
- [ ] **Error states styled** - Error messages and states have CSS
- [ ] **Both themes covered** - Dark and light theme styles

## Modal/Dialog Checklist
When adding modals:

- [ ] **Modal container styled** - Both themes
- [ ] **Modal content styled** - Both themes
- [ ] **Close button functional** - X button works
- [ ] **Click outside to close** - Event listener added (if applicable)
- [ ] **Escape key to close** - Event listener added (if applicable)
- [ ] **Focus management** - Auto-focus appropriate element
- [ ] **Form elements styled** - All inputs in both themes

## JavaScript Functionality Checklist

- [ ] **Event listeners attached** - All buttons/forms have handlers
- [ ] **Global functions accessible** - onclick handlers are global
- [ ] **Error handling added** - Try/catch where appropriate
- [ ] **Firebase sync** - saveToFirebase() called after changes
- [ ] **UI updates** - Render functions called after data changes
- [ ] **Cleanup functions** - Remove listeners when needed

## Testing Checklist

- [ ] **No console errors** - Check browser console
- [ ] **Dark theme tested** - Visual verification
- [ ] **Light theme tested** - Visual verification
- [ ] **Mobile responsive** - Test on small screen
- [ ] **Firefox compatibility** - Test in Firefox if available
- [ ] **Form submission works** - All forms submit properly
- [ ] **Data persists** - Firebase saving/loading works

## Common Mistakes to Avoid

1. **Missing light theme CSS** - Always add both themes
2. **Incomplete input type coverage** - Remember number, select, textarea
3. **No focus/hover states** - Interactive elements need visual feedback
4. **Global function not defined** - onclick handlers must be window-accessible
5. **Missing event listeners** - Buttons need click handlers
6. **No Firebase sync** - Data changes must saveToFirebase()
7. **Form not clearing** - Reset forms after submission
8. **Z-index issues** - Modals need proper layering

## Pre-Completion Validation

Before marking work complete:

1. **Run get_errors tool** - Check for syntax errors
2. **Review all files changed** - Verify completeness
3. **Check for TODO/FIXME** - No unfinished code
4. **Verify both themes** - CSS exists for both
5. **Test in browser** - Actually load and test the page

## Documentation Checklist

Before each git push:

- [ ] **README.md updated** - Document all new features/changes
- [ ] **No emojis in README** - Use plain text only, no emoji characters
- [ ] **Feature descriptions clear** - Explain what was added/changed
- [ ] **Usage instructions included** - How to use new features (if applicable)

---

## How to Use This Checklist

When asked to add new features:
1. Read this checklist BEFORE starting
2. Check off items as you implement
3. Review the "Common Mistakes" section
4. Run the "Pre-Completion Validation" steps
5. Only then report completion to user

When user reports missing functionality:
1. Review which checklist items were missed
2. Fix all related items, not just the reported one
3. Update checklist if new patterns emerge
